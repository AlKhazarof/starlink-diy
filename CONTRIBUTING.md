## ğŸ“‹ Getting Started

1. **Fork the Repository**: Create your own fork of the project
2. **Clone Your Fork**: `git clone https://github.com/your-username/starlink-diy.git`
3. **Create a Branch**: `git checkout -b feature/your-feature-name`
4. **Make Changes**: Implement your contribution
5. **Test Thoroughly**: Ensure your changes work as expected
6. **Commit**: Use clear, descriptive commit messages
7. **Push**: Push your changes to your fork
8. **Pull Request**: Submit a PR with a clear description

## ğŸ’» Development Guidelines

### Code Style
- **Python**: Follow PEP 8 style guide
- **C/C++**: Use consistent indentation (4 spaces)
- **Comments**: Write clear, concise comments explaining complex logic
- **Documentation**: Update relevant docs with code changes

### Commit Messages
Use clear, descriptive commit messages:
```
Add satellite tracking algorithm for LEO constellations

- Implement TLE parsing and orbit calculation
- Add Doppler shift correction
- Include unit tests for edge cases
```

### Pull Request Process
1. Update documentation to reflect your changes
2. Add tests if applicable
3. Ensure all tests pass
4. Update the README.md if needed
5. Reference any related issues in your PR description
